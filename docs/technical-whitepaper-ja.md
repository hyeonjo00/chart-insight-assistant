# Chart Insight Assistant 技術ホワイトペーパー

## 概要

Chart Insight Assistant は、チャートスクリーンショットを構造化されたシナリオベースの市場解釈に変換する Next.js アプリケーションです。このプロジェクトは、アップロード UI、サーバーサイド画像検証、OpenAI によるビジョン分析、構造化 JSON 出力、ブラウザローカル履歴、Google AdSense 対応構成を 1 つのプロダクトフローとして統合しています。

このシステムの目的は、確定的な売買シグナルを提供することではありません。ユーザーがチャートを考察するための慎重なシナリオ分析を提供し、将来の価格変動を保証しないように設計されています。

## プロダクト目標

このプロジェクトは、5 つの実用的な目標を中心に設計されています。

- チャートスクリーンショットを簡単にアップロードできるワークフローを提供する
- OpenAI API キーをサーバー側で安全に保護する
- 自由文ではなく予測可能な構造化データを返す
- トレーディング視点で素早く読める結果 UI を提供する
- 実際の SaaS プロダクトへ拡張しやすい小さなアーキテクチャを保つ

## システムアーキテクチャ

```text
Browser
  |
  | FormData で画像をアップロード
  v
Next.js App Router UI
  |
  | POST /api/analyze
  v
Next.js Route Handler
  |
  | ファイル検証、画像変換、OpenAI 呼び出し
  v
OpenAI Responses API
  |
  | 構造化 JSON
  v
結果カード + localStorage 履歴
```

アプリは App Router を使い、UI ルートとサーバーサイド API の動作を分離しています。アップロードと結果表示の体験は `components/chart-upload-panel.tsx` にあり、OpenAI 連携は `app/api/analyze/route.ts` に分離されています。

## フロントエンド設計

フロントエンドは Analyze ページを中心に構成されています。アップロードパネルは、ファイル選択、ドラッグアンドドロップ状態、プレビュー URL、ローディング状態、API エラー、分析結果、ローカル履歴保存を管理します。

主な設計判断:

- リクエスト前にクライアントでファイルを検証します。
- 有効なファイルが選択されるとすぐにプレビューを表示します。
- 有効な画像がない場合、Analyze ボタンは無効になります。
- ローディング UI はページ全体をブロックせず、スケルトン風の表示にします。
- 結果カードでは `Bias` と `Confidence` をバッジで強調します。
- 結果カードには常に免責文を表示します。

## API ルート設計

サーバールートは `multipart/form-data` でアップロードされたチャート画像を受け取ります。その後、OpenAI に送信する前にサーバー側で画像を再検証します。

サーバーの責務:

- `process.env.OPENAI_API_KEY` から API キーを読み取る
- 有効なアップロードファイルがないリクエストを拒否する
- 対応する画像 MIME タイプのみ許可する
- 基本的なファイルサイズ制限を適用する
- 画像を base64 data URL に変換する
- 画像とプロンプトを OpenAI に送信する
- 構造化 JSON 出力を要求する
- 返された JSON を検証してからクライアントに返す

この構造により、ブラウザに秘密鍵を公開せず、安全ルールをサーバー側の 1 箇所で管理できます。

## OpenAI プロンプト戦略

分析プロンプトは意図的に慎重に設計されています。モデルには、アップロードされたチャートスクリーンショットを分析し、有効な JSON のみを返し、断定的な表現を避け、チャートが不明瞭な場合は `neutral` bias と `low` confidence を使うよう指示しています。

この方針により、機能は売買シグナル販売ではなく、シナリオ分析プロダクトとして位置づけられます。

## 構造化出力契約

UI は API が次の形式を返すことを前提としています。

```json
{
  "bias": "long | short | neutral",
  "entry_zone": "string",
  "invalidation_zone": "string",
  "take_profit": ["string"],
  "confidence": "low | medium | high",
  "summary": "string"
}
```

フロントエンドはこのレスポンスを `Long`、`Short`、`Neutral`、`Low`、`Medium`、`High` などの表示用ラベルに変換します。

## ローカル履歴保存

完了した分析結果はブラウザの `localStorage` に保存されます。これにより、MVP 段階でアカウントやデータベースの複雑さを追加せずに履歴機能を提供できます。

保存フィールド:

- `id`
- `createdAt`
- `bias`
- `confidence`
- `summary`
- `entryZone`
- `invalidationZone`
- `takeProfitTargets`
- 任意の軽量画像プレビュー

履歴ヘルパーは保存件数を制限し、ブラウザストレージの使用量が大きくなりすぎないようにしています。

## AdSense 対応構成

プロジェクトは、コアのプロダクト体験を損なわずに収益化の準備を含んでいます。

AdSense 関連構成:

- `app/layout.tsx` でグローバルスクリプトを読み込み
- `components/ad-banner.tsx` の再利用可能な広告スロット
- サイト確認用の `public/ads.txt`
- レイアウトシフトを抑える安定した広告領域

実際の広告表示には、AdSense で発行された有効なスロット ID が必要です。

## セキュリティと信頼性

主な安全対策:

- OpenAI API キーはサーバー側にのみ存在します。
- ブラウザは OpenAI API キーを受け取りません。
- アップロードファイルはクライアントとサーバーの両方で検証されます。
- OpenAI 出力は既知の JSON 形式に制限されます。
- UI には投資助言ではないことを示す免責文が含まれます。
- 分析はシナリオベースであり、結果を保証しません。

## 現在の制限

- 履歴はブラウザ内にのみ保存されます。
- ユーザー認証はまだありません。
- 永続データベースはまだありません。
- rate limit はまだありません。
- ティッカー、取引所、時間足メタデータは自動抽出されません。
- 実際の AdSense スロット ID 設定が必要です。

## 結論

Chart Insight Assistant は、アップロード、検証、分析、構造化結果表示、履歴保存、デプロイ準備までの一連の AI プロダクトフローを小さなコードベースで実装しています。アーキテクチャは意図的にシンプルですが、プロダクト強化、収益化、長期的な拡張へ進むための道筋を明確に残しています。
